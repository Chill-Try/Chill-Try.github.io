目标
#

​ 一起动手写 Batch 和 VBScript
​ 让你的 Windows 10 系统获得夜间模式
​ 并配合任务计划程序，实现自动化


测试
#

本文所用环境是 win10
如果你是其它 windows 版本，请先进行以下测试：

WIN + R 打开运行，运行以下指令，它会对注册表值进行修改：

reg.exe add HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize /v AppsUseLightTheme /t REG_DWORD /f /d 0


运行后，观察一下 资源管理器、Chrome 浏览器 等，它们的窗口有没有变成暗色主题

将指令末尾的 0 改成 1，再次运行，观察这些窗口是否变成亮色主题

若没效果 骂微软并退出


写 NightMode.bat (手动切换夜间模式)
#

在 D:\ 根目录下，创建一个名为 .bat 的隐藏文件夹，用来存放我们的脚本

如果你不知道 如何在资源管理器中显示隐藏文件夹，百度，或把名字开头的点去掉

经过上面的测试，你已经知道，它是怎么工作的了

新建一个 NightMode.bat 文件，来写我们的脚本

思想是：
#

​ 先查询注册表 AppsUseLightTheme 的值

​ 它是亮色主题，就成设置暗色主题；是暗色主题，就设置成亮色主题

​ 因为 reg query 会返回三个值：键、类型、值，我们需要用 for 循环来取 值

:: D:\.bat\NightMode.bat

:: 例如返回 "AppsUseLightTheme REG_DWORD 0x1" 时，我们要取第三个值，空格作分隔符
for /f "tokens=3 delims= " %%i in ('
    reg query ^
    "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize" ^
    /v "AppsUseLightTheme"
') do (
    if 0x1==%%i (
        reg.exe add ^
        HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
        /v AppsUseLightTheme /t REG_DWORD /d 0 /f
        reg.exe add ^
        HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
        /v SystemUsesLightTheme /t REG_DWORD /d 0 /f
    ) else (
        reg.exe add ^
        HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
        /v AppsUseLightTheme /t REG_DWORD /d 1 /f
        reg.exe add ^
        HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
        /v SystemUsesLightTheme /t REG_DWORD /d 1 /f
    )
)
:: 这是注释，以免你忘记：1 亮色主题，0 暗色主题

只是简单的取反操作，我们可以看到，以上代码有重复冗余的部分


稍作优化：
#

​ 为了可以在 for 循环中，动态更新变量值
​ 我们需要在开头添加 setlocal enabledelayedexpansion，batch 特色

​ 目标是查询后，取反并置值
​ 因为只有 0 或 1 两种情况，我这里使用 1 - foo 来取反

​ 如果直接使用 ~foo 取反，实际上是将 0x0000 0001 转换为 0x1111 1110，即 -2
​ 当然，你也可以用 ~foo + 2 来实现 ，尽最大可能剥削压榨自己的电脑

:: D:\.bat\NightMode.bat

setlocal enabledelayedexpansion

for /f "tokens=3 delims= " %%i in ('
    reg query ^
    "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize" ^
    /v "AppsUseLightTheme"
') do (
    set /a i=1-%%i
    reg.exe add ^
    HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
    /v AppsUseLightTheme /t REG_DWORD /d !i! /f
    reg.exe add ^
    HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
    /v SystemUsesLightTheme /t REG_DWORD /d !i! /f
)
:: 这是注释，以免你忘记：1 亮色主题，0 暗色主题

写 Auto_Nightmode.bat (准备自动化)
#

脚本是我几年前写的，现在看这个命名有点难受，大小写 下划线 混用
但是 DOS 对大小写不敏感
你可以起一个 aUtO_niGHt_mOdE.bAT ，看起来很 Fashion

下面还有更特色的

这个脚本的功能是根据时间设置主题

晚上开夜间模式（暗色主题）
白天关夜间模式（亮色主题）

先判断时间，根据目前所处时间段，跳转执行命令

batch 特色：if 条件语句没有 与 或 非：

:: D:\.bat\Auto_Nightmode.bat

:: 获取时、分
set h=%time:~0,2%
set m=%time:~3,2%

:: 夜晚时间段 18:30~08:30
if %h% GEQ 0 (
	if %h% LEQ 8 (
		:: （可以将上行 LEQ 改成 LSS，省略下行 if，并将 EQU 8 那个 if 扔到外面。懒得改了）
		:: 00:00~08:00
		if %h% NEQ 8 (
			goto Night
		)
		:: 08:00~08:30
		if %h% EQU 8 (
			if %m% LSS 30 (
				goto Night
			)
		)
	)
)
if %h% GEQ 18 (
	:: 19:00~00:00
	if %h% NEQ 18 (
		if %h% LSS 24 (
			goto Night
		)
	)
	:: 18:30~19:00
	if %h% EQU 18 (
		if %m% GTR 29 (
			goto Night
		)
	)
)


:: 白天时间段 08:30~18:30
if %h% GEQ 8 (
	if %h% LEQ 18 (
		if %h% NEQ 8 (
			:: 18:00~18:30
			if %h% EQU 18 (
				if %m% LSS 30 (
					goto Day
				)
			)
			:: 9:00~18:00
			if %h% NEQ 18 ( 
				goto Day
			)
		)
		:: 8:30~9:00
		if %h% EQU 8 (
			if %m% GEQ 30 (
				goto Day
			)
		)
	)
)


:Night
echo %h% : %m% It's night
reg.exe add ^
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
/v AppsUseLightTheme /t REG_DWORD /d 0 /f
reg.exe add ^
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
/v SystemUsesLightTheme /t REG_DWORD /d 0 /f
exit

:Day
echo %h% : %m% It's day
reg.exe add ^
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
/v AppsUseLightTheme /t REG_DWORD /d 1 /f
reg.exe add ^
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize ^
/v SystemUsesLightTheme /t REG_DWORD /d 1 /f
exit

同样的，这里也有冗余
你可以像上面那样优化代码，将跳转语句改为设置变量值
这里刚好能学到 goto 的用法 （懒得改了）

此时配合任务计划程序，在正确时机运行它，就已经可以实现自动化了

但在进行任务计划程序的设置之前，要加一步 VBScript 来让它后台运行


VBScript 使脚本后台运行
#

为什么？

如果你在 Windows 上部署过 mysqld
你会发现它每天 00:00 都会弹个 cmd.exe 窗口来执行 Batch Script
此时全屏游戏窗口会被它挤出来

为实现如题所示的目标，我们创建 VBScript 脚本 RunAuto_Nightmode.vbs

它就更简单了，只是让我们写的 Batch Script 后台运行：

' D:\.bat\RunAuto_Nightmode.vbs

Set shell = WScript.CreateObject("WScript.Shell")
Set fso = WScript.CreateObject("Scripting.FileSystemObject")

' 设置当前工作目录为脚本所在的目录
shell.CurrentDirectory = fso.GetParentFolderName(WScript.ScriptFullName)

' 使用相对路径运行批处理文件
run = shell.Run("Auto_Nightmode.bat", 0)

任务计划程序（自动执行）
#

最后一步，在自动执行执行 VBScript

帮你回忆一下，当你的 aUtO_niGHt_mOdE.bAT

  • 在 08:30 ~ 18:30 时间段运行时，关闭夜间模式
  • 在 18:30 ~ 08:30 时间段运行时，开启夜间模式

（他们不是同一个时间段，因为我们的 24 小时 “嘴里含着自己的脚” 是一条衔尾蛇）

虽然你可以周期性的，每过几分钟就执行它，它只是一个小脚本而已
但后台周期运行，多少都会损失一点性能

所以我们要找到正确执行时机

找出正确时机：
#

分析过后可得出结论：

​ 每天 08:30 和 18:30 这两个时间点执行 （废话）

似乎这样就可以了，但这里有个小坑
当你电脑处于 关机、睡眠、休眠，甚至是 锁定 状态下
任务计划程序都不会按计划执行

不难发现，从任何状态中恢复时，都需要解锁电脑

所以，正确执行时机：

  1. 每天 08:30 和 18:30 这两个时间点
  2. 解锁工作站时

Chrome 浏览器插件
#

浏览器我们经常用到

多数程序会自动检测 Windows 的系统主题

当系统主题改变时，它们会自动切换自身主题


Chrome 也是如此，设置一下即可（一般默认即跟随系统）

你运行 NightMode.bat 脚本后，我的博客也会随 Chrome 一起切换主题

但并不是所有网站都支持夜间模式


所以这里推荐一个自己在用的 浏览器插件 Dark Reader

Edge 和 Chrome 一个内核，也支持

它可以对绝大多数网站提供夜间模式支持

你可以设置插件，让它跟随浏览器主题

而浏览器又跟随系统主题

系统主题改变 -> Chrome 主题改变 -> 触发插件 -> 网页主题改变
